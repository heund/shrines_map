## Current Delivery Model

The map is built as a single self-contained HTML file.

Key characteristics:

- No frontend build tools
- No framework
- No application server required beyond static file serving
- All project data is embedded inline at generation time
- The page is intended to be portable as a standalone document

Although it is self-contained, it should still be served through a local or remote HTTP server for reliable browser behavior.

## Map Provider

The current implementation uses Google Maps JavaScript API.

Reason for current provider:

- Kakao Maps browser-side loading was rejected under the local development workflow
- Google Maps proved easier to run under the current project constraints
- Korean presentation is preserved through explicit localization settings

Google Maps API is loaded with:

- `language=ko`
- `region=KR`

The generator reads:

- `GOOGLE_MAPS_API_KEY`

from:

## Overall Layout

The page uses a two-part layout:

- top header
- main content area with sidebar + map

### Header

The header contains:

- title: `제주비엔날레 장소 지도`
- subtitle describing the three compared layers
- a compact count pill showing total records
- a mobile sidebar toggle button

### Main Content

The content area is split into:

- left control sidebar
- right map canvas

Important behavior:

- the page itself does not scroll
- the map fills the available viewport height
- only the sidebar scrolls internally

This was intentionally changed so the map stays visually fixed and the control panel behaves as the scrollable area.

## Sidebar Structure

The sidebar currently appears in this order:

1. `비엔날레 장소`
2. `신당`
3. `문화유산`
4. `기본 지도`

### 1. Biennale Venues Section

This section includes:

- section total count
- `전체 표시` master checkbox
- a list of all venue names as clickable items
- a short explanatory note

The venue names are individually listed and can be clicked to:

- clear old-city focus mode
- recenter the map to that venue
- zoom to a closer venue-level view

There are no venue sub-checkboxes at present.

### 2. Shrines Section

This section includes:

- section total count
- `전체 표시` master checkbox
- ownership sub-checkboxes
- a note showing how many shrines have no coordinates

Ownership groups are:

- `국가`
- `개인`
- `공동`
- `법인`
- `미상`

Behavior:

- coordinate-less shrines are not plotted
- they remain counted in the section total

### 3. Heritage Section

This section includes:

- section total count
- `전체 표시` master checkbox
- subtype checkboxes based on `ccmaName`
- an expandable object-type list based on `gcodeName`

Primary visible grouping:

- `ccmaName`

Secondary reference grouping:

- `gcodeName`

The user can filter the map by heritage designation type while still seeing the full object-type breakdown in the expandable list.

### 4. Base Map Section

This section controls Google’s built-in map landmarks, not the project’s own layers.

It includes:

- `지도 랜드마크 표시`

When this toggle is off, the interface suppresses:

- Google POI landmarks
- transit station labels
- road label icons
- highway road labels

This was added so built-in Google map labels do not overwhelm the project layers.

## Marker Design

Three distinct marker families are used.

### Heritage Markers

Shape:

- diamond

Purpose:

- distinguish heritage from both venues and shrines

Current size:

- smaller than the venue markers
- intentionally reduced from the earlier version to prevent visual crowding

### Venue Markers

Shape:

- star

Purpose:

- strong visual emphasis for biennale sites

Current color:

- bright yellow

This was changed from a cooler teal/blue to a more prominent yellow for visibility.

### Shrine Markers

Shape:

- custom shrine/pin-like symbol

Purpose:

- make shrines immediately distinguishable from heritage diamonds and venue stars

This replaced an earlier circle marker because the circles were too difficult to separate at a glance.

## Old-City Focus Mode

One of the major interaction systems in the current interface is the Jeju old-city focus mode.

### Button Label

The button now behaves as a toggle:

- default state label: `원도심 포커스`
- active focus state label: `전체`

### Focus Mode Behavior

When `원도심 포커스` is activated:

- old-city focus mode is enabled
- the floating focus description card disappears
- the old-city bounding rectangle becomes visible
- heritage markers outside the focus bounds are hidden
- shrine markers outside the focus bounds are hidden
- venue markers remain visible even outside the focus bounds

This venue exception was intentional and requested explicitly.

### Exiting Focus Mode

When the button shows `전체` and is pressed:

- focus mode is disabled
- the bounding rectangle is hidden
- the floating focus description card returns
- all filtered-out heritage and shrine markers return

### Important Interaction Rule

Focus mode no longer auto-reverts when the map is moved.

Earlier versions incorrectly did this:

- dragging the map reset focus mode
- the button text reverted automatically

That behavior was removed so focus mode only changes when explicitly toggled through the button.

### No Auto-Zoom on Focus

Earlier versions also performed a map zoom/fit action when focus mode was activated.

That behavior caused two problems:

- it appeared to zoom into the center instead of the intended area
- it interfered with the visual filtering timing

Current behavior:

- the focus button no longer changes zoom
- it only changes filtering state and visibility rules

### Bounding Rectangle Styling

Current behavior:

- the rectangle is outline-only
- no tinted district-style fill overlay is shown
- the rectangle is hidden by default
- it appears only while focus mode is active

## Layer Visibility Logic

### Section Master Toggles

Each major section has a `전체 표시` master checkbox.

Current behavior:

`문화유산`

- turning master off clears all heritage subtype checkboxes
- turning master on restores all heritage subtype checkboxes

`신당`

- turning master off clears all shrine ownership checkboxes
- turning master on restores all shrine ownership checkboxes

`비엔날레 장소`

- no child checkbox set exists
- master toggle directly controls venue visibility

### Child Filter Reactivation

This was an important bug fix.

Earlier problem:

- if `전체 표시` was turned off for a section
- and then a single child checkbox was turned back on
- the layer still stayed invisible because the parent section state remained off

Current behavior:

- if a user re-checks a child option in heritage or shrines
- the section automatically becomes active again
- only the selected child subset is shown

Examples:

- `신당 -> 개인` alone now displays only private shrines
- one heritage subtype alone now displays only that subtype

### Partial Master State

For `문화유산` and `신당`, the master checkbox now supports a partial visual state:

- fully checked when all children are selected
- unchecked when none are selected
- indeterminate when some but not all are selected

## Venue-Specific Behavior

### Venue Count

The venue dataset contains 5 venues.

When fewer than 5 seemed visible, the main reasons were:

- map view framing
- old-city focus mode

This was addressed through:

- explicit venue list in the sidebar
- slightly wider default zoom
- venue persistence during old-city focus mode

### Venue Name List

Each venue name is rendered as its own clickable row.

Click behavior:

- disables focus filtering
- pans to the venue
- zooms to a closer venue-level scale

## Base Map Landmark Suppression

The Google base-map toggle is designed to suppress built-in map clutter in one action.

Current suppression set includes:

- `poi`
- `transit.station`
- `road labels.icon`
- `road.highway labels`

This was extended specifically so road numbers such as national route labels are also reduced together with Google landmarks.

## Mobile Behavior

On smaller screens:

- the sidebar becomes an off-canvas panel
- the header `필터` button toggles sidebar visibility
- the map still fills the main viewport area

The page should be opened via local HTTP serving, not directly as a `file://` document.

Example:

```powershell

```

Then open:

```text
http://localhost:8000/index.html
```

## Summary of Major Completed Decisions

The current interface reflects the following finalized decisions:

- Google Maps is the active provider
- Korean map localization is enabled
- the page is a single self-contained HTML output
- the sidebar order is:
  - 비엔날레 장소
  - 신당
  - 문화유산
  - 기본 지도
- venue markers are bright yellow stars
- heritage markers are smaller, bright diamonds
- shrine markers use a distinct shrine/pin-like shape
- coordinate-less shrines are never plotted
- Jeju old-city focus is a toggle, not a one-off button
- venues remain visible during old-city focus
- the old-city box appears only during active focus mode
- sidebar scrolling is isolated from page scrolling
- master toggles now synchronize with child toggles
- child toggles can reactivate their parent section after master deselection
- Google built-in landmarks can be suppressed independently of project layers
